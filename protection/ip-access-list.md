# IP 黑白名单

IP 黑白名单用于按来源 IP 或 CIDR 对请求做允许或阻断。它是独立模块，不与路径、Header、Host 访问控制混用。

## 适用场景

- 放行办公网、监控节点或可信回源地址。
- 阻断已确认攻击源、代理池或异常网段。
- 在应用级范围内处理某个业务的特殊来源策略。

## 后台入口

进入后台“IP 黑白名单”。创建条目时选择允许或阻断、IP 或 CIDR、全局或指定应用范围。

## 如何配置

1. 在“IP 黑白名单”页面选择作用范围：全局或指定防护应用。
2. 选择动作：允许或阻断。
3. 选择地址类型：Exact IP 或 CIDR。Exact IP 填写单个来源地址，CIDR 填写网段。
4. 填写 IP、CIDR 前缀长度、名称和启用状态。
5. 保存后进入发布预览，确认 allow、block、Exact IP、CIDR、全局和应用级数量。

## 匹配范围

精确 IP 只匹配单个来源地址。CIDR 匹配网段，例如 `203.0.113.0/24`。Gateway 会基于有效客户端 IP 判断，是否能拿到真实 IP 取决于可信代理配置。

站点或应用级条目优先于全局条目；同一范围和精度下允许条目优先于阻断条目，便于应急恢复。精确条目和 CIDR 条目分别参与判断。

## 发布后生效

保存 IP 条目后必须发布新版本。Gateway 使用发布后的 `ip_access_index` 本地索引执行，不查询控制面数据库。

## 验证方式

1. 使用可控来源 IP 或可信代理测试头访问目标应用。
2. 确认允许条目正常代理，阻断条目返回拒绝响应。
3. 在发布预览中确认 enabled、allow、block、exact IP、CIDR、global、site-scoped 数量。

## 日志排查字段

在拦截 / 拒绝记录和访问日志中查看来源 IP、处置结果和拒绝原因。命中 WAF 事件时关注 `module=ip-access-list`、动作和条目名称。

## 误伤和风险

- 全局 CIDR 阻断影响面大，建议优先应用级。
- 真实 IP 配置错误会导致误放行或误阻断。
- 不要用 IP 黑白名单表达路径规则；路径、Header、Host 使用访问控制。
