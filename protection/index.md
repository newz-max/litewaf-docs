# 防护模块总览

LiteWaf 的防护模块面向后台操作员组织常见 WAF 能力。控制面保存模块配置，Gateway 只执行已发布版本；新增、编辑、删除、启停规则后都需要发布配置才会影响真实流量。

## 模块入口

| 模块 | 适用场景 |
| --- | --- |
| [CC 防护](cc-protection.md) | 高频访问、登录爆破、API 调用滥用、404 扫描 |
| [攻击防护](attack-protection.md) | SQL 注入、XSS、RCE、路径穿越等托管攻击规则 |
| [IP 黑白名单](ip-access-list.md) | 按来源 IP 或 CIDR 做全局或应用级允许、阻断 |
| [访问控制](access-control.md) | 按路径、Header、Host 做允许、观察、阻断 |
| [上传防护](upload-protection.md) | 上传路径、危险扩展名、上传大小限制 |
| [Bot / 人机验证](bot-protection.md) | JS challenge、本地 captcha、行为评分、设备信号 |
| [动态防护 / 等候室](dynamic-protection.md) | 动态令牌、页面动态化、本地等候室 |
| [高级规则生态](advanced-rules.md) | 规则包导入、签名状态、规则测试、社区规则流转 |

## 通用上线流程

1. 在对应模块新增或编辑规则，先选择观察或窄范围路径。
2. 使用发布预览检查将要下发到 Gateway 的规则数量、动作和风险提示。
3. 发布配置。
4. 通过网关访问目标路径，确认正常请求和命中请求的响应。
5. 在访问日志、攻击日志、拦截 / 拒绝记录和统计报表中确认模块字段。

## 匹配范围

CC 防护的“匹配方式”下拉框支持“前缀”“精确”“Glob”。访问控制、上传防护、Bot / 人机验证和动态防护主要使用“前缀”“精确”。括号中的 `prefix`、`exact`、`glob` 只用于说明发布配置中的底层值。攻击防护和高级规则按规则目标字段执行，IP 黑白名单按来源 IP 或 CIDR 执行。

## 发布边界

模块页面中的保存、预览、测试、导入、同步或导出操作不会直接改变 Gateway 活动配置。只有发布新版本后，Gateway 才会加载新的本地配置并在请求热路径执行。

## 验证入口

- 访问日志：确认请求是否到达 Gateway，查看状态码和处置结果。
- 攻击日志：按 `module` 过滤具体模块事件。
- 拦截 / 拒绝记录：从被阻断或被拒绝请求回跳到模块原因。
- 发布记录：确认当前生效版本、发布摘要和风险提示。
