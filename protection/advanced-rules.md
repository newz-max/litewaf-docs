# 高级规则生态

高级规则生态用于规则包导入、签名状态、来源追踪、规则测试和社区贡献流转。Gateway 只读取已发布的可执行规则，不在请求热路径拉取远程目录或校验签名。

## 适用场景

- 导入本地 JSON 规则包。
- 预览外部规则包的新增、变更、跳过和无效规则。
- 对阻断型规则先运行测试，再绑定策略发布。
- 准备社区规则贡献和导出。

## 后台入口

进入后台“高级规则生态”。使用预览、导入、测试、导出等操作管理规则包。

## 推荐配置

| 场景 | 建议 |
| --- | --- |
| 首次导入 | 先预览，确认签名、来源、规则数量和无效项 |
| 阻断规则上线 | 先测试，再使用观察或低风险策略发布 |
| 社区规则维护 | 保留来源、分类、攻击类型、规则组和说明 |
| 规则导出 | 只导出脱敏、可公开的规则元数据和表达式 |

## 匹配范围

导入后的规则复用现有规则字段，例如目标、表达式、动作、分值、模块、分类、攻击类型、规则组、优先级和启用状态。规则是否执行取决于发布后的策略绑定和模块配置。

规则测试使用受限样例请求评估表达式，支持路径、Query、Header、Body 摘要和上传元数据，不保存完整 Body、Authorization、Cookie 或上传文件内容。

## 发布后生效

预览、同步、更新预览、导出预览和规则测试都不会自动启用、禁用、删除或发布规则。只有导入或显式应用后，控制面规则数据才会变化；仍需发布后 Gateway 才会执行。

## 验证方式

1. 对规则包执行预览并处理无效项。
2. 导入后在规则管理中确认真实规则存在。
3. 使用规则测试验证命中。
4. 绑定策略并发布，在攻击日志确认命中事件。

## 日志排查字段

按规则所属模块查询，例如 `attack-protection` 或高级表达式规则事件。关注 `rule_id`、`rule_name`、`target`、`action`、`score`、`summary`、`disposition` 和包来源摘要。

## 误伤和风险

- 未签名规则包可以本地导入，但发布前需要更严格审查。
- 不要把未经测试的阻断规则直接发布到全站。
- 远程目录不可用不影响已发布规则执行。
