# 规则编写

LiteWaf 规则用于描述请求检测条件和处置方式。规则应保持可解释、可回滚，并避免依赖远程执行逻辑。

## 基础字段

| 字段 | 说明 |
| --- | --- |
| 名称 | 规则的可读名称 |
| 目标 | URI、Header、Query、Body、上传元数据等检测目标 |
| 表达式 | 匹配条件 |
| 动作 | 记录、拦截或计分 |
| 分值 | 参与策略阈值判断 |
| 状态 | 是否启用 |

## 编写建议

- 先用记录或低风险策略验证命中情况，再切换到拦截。
- 为误报风险较高的规则设置清晰名称和说明。
- 将通用规则和业务专用规则分开维护。
- 发布前使用预览确认规则会进入目标站点策略。

如果需要按后台模块配置规则，先查看 [防护模块总览](../protection/)。CC 防护、攻击防护、访问控制、上传防护、Bot / 人机验证、动态防护和高级规则生态的匹配范围并不完全相同。

## 不支持的边界

当前阶段不执行第三方远程代码，不把任意 Lua 插件作为社区规则动态运行，也不在 Gateway 请求热路径进行远程签名校验。

详细字段和示例见 [`litewaf-api/doc/规则编写指南.md`](https://github.com/newz-max/litewaf-api/blob/master/doc/%E8%A7%84%E5%88%99%E7%BC%96%E5%86%99%E6%8C%87%E5%8D%97.md)。
