# 日常使用

LiteWaf 的日常操作围绕“创建资源、调整防护、发布配置、验证网关效果”展开。控制面保存配置，Gateway 只执行已发布版本。

## 操作流程

1. 创建站点，填写域名、监听端口、协议和源站。
2. 配置规则、策略、IP 黑白名单、访问控制、限流和防护模块。
3. 使用发布预览检查将要下发到 Gateway 的配置。
4. 发布配置。
5. 通过网关访问业务域名或端口，查看访问日志、攻击日志、统计报表和发布记录。

## 发布生命周期

以下变更必须发布后才会进入 Gateway：

- 站点、监听端口、证书和源站。
- 规则、规则集、策略和评分阈值。
- IP 黑名单、IP 白名单、访问控制和限流。
- CC 防护、Bot 防护、动态防护、上传防护和高级规则配置。

如果后台保存成功但网关行为没有变化，优先检查发布记录、当前生效版本和 Gateway 日志。

## 验证方式

| 目标 | 建议检查 |
| --- | --- |
| 正常代理 | 通过 Gateway 访问业务 URL，确认返回源站响应 |
| WAF 拦截 | 构造命中规则的请求，确认返回拦截响应 |
| 日志回流 | 查看访问日志、WAF 事件和统计报表 |
| 发布状态 | 查看发布记录、版本号和 Gateway 配置刷新状态 |

完整操作细节见 [`litewaf-api/doc/使用说明.md`](https://github.com/newz-max/litewaf-api/blob/master/doc/%E4%BD%BF%E7%94%A8%E8%AF%B4%E6%98%8E.md)。
