API 概览 ​

LiteWaf 管理 API 使用 /api/v1 前缀，响应格式为 JSON。Dashboard 通过该 API 管理站点、规则、策略、发布、日志、观测和系统设置。

鉴权 ​

  • 登录接口返回 Bearer Token。
  • 管理接口按角色权限校验。
  • 审计日志记录关键变更。

资源范围 ​

资源说明
站点和监听域名、协议、端口、源站和证书绑定
规则和策略检测目标、表达式、动作、评分和阈值
发布发布预览、发布记录、版本回滚
IP 名单和访问控制独立 IP 黑白名单、访问控制和限流配置
日志和观测访问日志、攻击日志、统计报表和指标
运维健康检查、版本、备份和恢复

集成建议 ​

集成方应优先按“保存配置 -> 发布预览 -> 发布 -> 验证日志”的顺序调用 API。只保存配置不会改变 Gateway 当前执行版本。

完整接口字段和示例见 litewaf-api/doc/API文档.md。
